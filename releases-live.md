# MarlinSpike Live — Releases

The web UI and Docker orchestration layer for MarlinSpike (`app.py`, templates, Docker).

| Version | Date | Changes |
|---------|------|---------|
| v2.0.0 | 2026-03-27 | **Release 2.0.0 — live analyst flow validated.** Deployed the analyst workspace refresh with fast/full scan profiles, chunk-aware file-based chain execution for large captures, improved stop semantics, and preserved MITRE sidecar generation on successful full-chain reports. Release smoke-tested end to end against `marlinspike.riverrisk.io` with Playwright using login -> dashboard -> projects -> preset scan -> scans -> viewer -> assets. |
| v1.5.0 | 2026-02-23 | **Projects system + upload pipeline overhaul.** Project-based file/report organization: `Project` model with per-user unique names, Default project auto-created on migration. Projects page (`/projects`) with two-column browser: sidebar project list, tabbed content (PCAPs + upload/scan, Reports + compare/diff, History). Dashboard project selector dropdown with inline create. Upload pipeline hardened: 500 MB limit, PCAP magic-byte validation (pcap LE/BE + pcapng), auto-slice to 100 MB via editcap for processing, silent zstd-compressed archival to `data/submissions/` (TOS compliance, background thread). Client-side size check before upload. Directory structure now `data/{uploads,reports}/<user_id>/<project_id>/`. Idempotent startup migration moves flat files into Default project. Project CRUD API (list/create/rename/delete + per-project files/reports/upload). All existing endpoints accept `?project_id`. Scan history includes `project_id`/`project_name`. System page shows submissions archive stats. `zstandard` dependency added |
| v1.4.0 | 2026-02-23 | **Engine v1.4.0 — Five-tuple port analysis + C2 detection.** Viewer: edge hover tooltips show port labels, node detail panel shows service ports with category color tags, port labels on connection lines. Report pane: Port Analysis table (port/protocol/category/connections/peers), C2 Indicators panel (beacon confidence bars, DNS entropy display, severity badges). Assets page: service ports column with category tags, port labels on connections, CSV export includes service ports. `NO_AUTH_OBSERVED` finding suppressed from per-asset risk displays (still in report summary) |
| v1.3.0 | 2026-02-23 | Assessment Report pane in viewer: collapsible bottom drawer with executive summary, risk findings with remediation recommendations, detailed asset inventory cards (identity, protocols, connections, per-node risks), protocol breakdown, Purdue violations. Google OAuth sign-in (authlib): `/login/google` + `/auth/google/callback`, auto-create user from email, graceful degradation when unconfigured. User model extended with `email` + `oauth_provider` columns |
| v1.2.1 | 2026-02-23 | Engine v1.2.1 deployed — tshark L2 field name fix (Stage 2 no longer fails on LLDP/CDP/LACP fields) |
| v1.2.0 | 2026-02-23 | Topology diff viewer (`/api/reports/diff`): side-by-side SVG comparison of two reports with added/removed/changed node highlighting. Live scan viewer (`/api/runs/<id>/live`): real-time topology updates during active captures. Report list checkbox selection for diff. Engine v1.2.0 |

| v1.0.0 | 2026-02-22 | Initial deployment release. Flask web app with PostgreSQL auth, scan orchestration (PCAP + live capture), concurrent run management, run status/output streaming, report download/delete, network interface enumeration. Docker Compose (app + postgres:16-alpine). Engine v1.0.0 |

## Deployment

- **Stack**: Docker Compose — `marlinspike-app` (Python 3.12-slim + tshark) + `marlinspike-db` (postgres:16-alpine)
- **Port**: `127.0.0.1:5001` by default
